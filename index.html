Text-to-SQL with Guardrails

Natural language to SQL with a zero-write guardrail layer, a read-only sandbox, and a back-translation detector that catches SQL answering the wrong question. 10/10 on the eval suite, zero unsafe queries.

Architecture

How a request moves through the system.

1
Schema-aware prompting

The database is introspected into tables, columns, foreign keys, and sample values; a relevance filter keeps only the tables the question needs.

2
Structured generation

The generator returns SQL plus an explanation, confidence, and the tables it touches.

3
Guardrail middleware

DDL and DML blocked, single statement only, LIMIT enforced, subquery depth capped; every block is logged with the rule that fired.

4
Read-only sandbox

Execution happens on a read-only connection, a second line of defense under the guardrails.

5
Hallucination detection

The SQL is translated back into a question and compared with the original; low alignment or multi-query disagreement flags the result instead of returning it silently.

6
Confidence score

Syntax validity, back-translation alignment, sanity checks, and query agreement combine into one number shown with every result.

How to use

Runs fully offline with a deterministic provider; add API keys to switch to live models.

# offline: seeded commerce DB + eval suite
pip install -r requirements.txt
python demo.py
# -> 10/10 cases behaved as expected; 0 unsafe queries executed

uvicorn src.text2sql.api:app --reload
curl -X POST localhost:8000/v1/query \
  -d '{"question": "revenue by country"}'
